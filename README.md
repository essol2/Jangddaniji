# Jangddanji

---

## 여정 다이어리 영상 기능 기획

### 개요

하루 여정을 진행하는 동안 1시간마다 2초짜리 영상 클립을 촬영하고, 여정이 완료되는 시점에 모든 클립을 하나의 영상으로 합쳐 여정의 흐름을 담은 타임랩스 다이어리 영상을 자동으로 생성한다.

---

### 사용자 플로우

```
여정 시작
    ↓
매 정시(ex. 11:00, 12:00, 13:00...)마다
로컬 알림 발송 → "여정 기록 시간입니다! 탭하여 영상 촬영"
    ↓
사용자가 알림 탭 → 앱 포그라운드 전환
    ↓
카운트다운(3-2-1) 후 카메라 자동 2초 녹화 시작
    ↓
녹화 완료 → 클립 로컬 저장 (DayRoute와 연결)
    ↓
(반복 — 매 정시마다)
    ↓
사용자가 오늘 여정 "완료" 버튼 탭
    ↓
촬영된 클립이 1개 이상이면 영상 합산 작업 시작
    ↓
각 클립에 촬영 시각 텍스트 오버레이 추가 (ex. "11:00")
클립들을 시간순으로 이어 붙여 최종 영상 생성
    ↓
"영상 보러가기" 버튼 노출
    ↓
사용자가 영상 확인 및 다운로드 (사진 앨범 저장)
```

---

### 영상 클립 스펙

| 항목 | 내용 |
|------|------|
| 촬영 주기 | 매 정시 1회 (ex. 09:00, 10:00, 11:00...) |
| 클립 길이 | 2초 |
| 시각 오버레이 | 화면 중앙, 굵은 흰색 볼드 텍스트 (ex. `11:00`) |
| 클립 전환 | 컷 편집 (별도 전환 효과 없음) |
| 저장 위치 | 앱 Documents 디렉터리, DayRoute ID 폴더 |
| 최종 영상 포맷 | MP4 (H.264) |

---

### 화면 구성

#### 촬영 화면 (알림 탭 시 표시)
```
┌─────────────────────┐
│                     │
│   카메라 프리뷰      │
│                     │
│   11:00 촬영 시간   │  ← 현재 정시 안내 텍스트
│                     │
│       [ ● ]         │  ← 촬영 버튼 (탭하면 2초 녹화 시작)
│                     │
└─────────────────────┘
```
- 알림 탭 → 카메라 프리뷰 화면 표시
- 사용자가 촬영 버튼(● ) 탭 → 2초간 녹화 자동 진행 (2초 후 자동 완료)
- 녹화 중 버튼 비활성화 + 남은 시간 표시 (ex. `1.5초`)
- 녹화 완료 → "11시 클립이 저장됐습니다" 토스트 표시 후 이전 화면으로 복귀

#### 여정 완료 화면 (기존 화면 하단 추가)
```
[영상 보러가기]  ← 클립 1개 이상일 때만 표시
```

#### 일자별 기록 화면 (기존 DayDetailView 하단 추가)
```
┌─────────────────────┐
│  오늘의 여정 영상    │
│  ┌───┐ ┌───┐ ┌───┐  │
│  │11:│ │13:│ │15:│  │  ← 촬영된 클립 썸네일 가로 스크롤
│  │00 │ │00 │ │00 │  │
│  └───┘ └───┘ └───┘  │
│  [▶ 영상 보러가기]   │  ← 합산 영상 재생 (클립 1개 이상 시)
└─────────────────────┘
```
- 클립 썸네일 가로 스크롤로 표시 (각 썸네일에 촬영 시각 표시)
- "영상 보러가기" 버튼: 합산 영상 재생 화면으로 이동
- 합산 영상 생성 중이면 ProgressView 표시

#### 영상 재생 화면
```
┌─────────────────────┐
│                     │
│   AVPlayer 영상     │
│                     │
└─────────────────────┘
     [다운로드]        ← 사진 앨범 저장
```

### 기술 스택

| 역할 | 기술 |
|------|------|
| 카메라 녹화 | `AVCaptureSession` + `AVCaptureMovieFileOutput` |
| 시각 오버레이 | `AVVideoCompositionCoreAnimationTool` + `CATextLayer` |
| 클립 합산 | `AVMutableComposition` + `AVAssetExportSession` |
| 영상 재생 | `AVPlayer` + `VideoPlayer` (SwiftUI) |
| 로컬 알림 | `UNUserNotificationCenter` |
| 사진 앨범 저장 | `PHPhotoLibrary` |

### iOS 기술 제약 및 대응

#### 백그라운드 카메라 제한
iOS는 앱이 백그라운드 상태일 때 카메라 접근을 허용하지 않는다.

**대응 방식:** 로컬 알림 기반 수동 촬영
- 매 정시에 로컬 알림 발송
- 사용자가 알림을 탭하면 앱이 포그라운드로 전환되어 카메라 화면 표시
- 사용자가 촬영 버튼 탭 → 2초 자동 녹화 완료 → 클립 저장

#### 알림을 놓쳤을 경우
- 해당 시간대 클립 없이 다음 정시 알림으로 넘어감
- 최종 영상은 촬영된 클립만으로 구성 (빈 시간대 생략)

#### 영상 합산 처리 시간
- 클립 수가 많을수록 내보내기 시간 증가 (`AVAssetExportSession` 비동기 처리)
- 완료 화면에서 진행 상태 표시 (ProgressView)
- 합산 완료 전 "영상 보러가기" 버튼 비활성화
